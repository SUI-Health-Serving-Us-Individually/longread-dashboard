All Annotated Variants — Full Genome (…)

Every record from the VEP-annotated VCF. This view runs SQL queries against a Parquet sidecar in the browser using DuckDB-WASM — no server, no full data load. Type to filter; click a column header to sort. Filters AND together. Use this to look up specific genes, positions, or HGVS notations, or to browse the entire genome interactively without the curated filters.

Why do 0/0 and ./. rows appear? This table holds the caller's complete output, not just confident variant calls. A caller such as DeepVariant emits a record at every site it evaluated and tags each with a FILTER:
  • PASS — a real variant in this sample (genotype 1/1, 0/1, 1|0, 1/2, …).
  • RefCall → genotype 0/0 — the site looked variant-like (e.g. a known polymorphic position or a few mismatched reads), but the caller decided this sample is homozygous reference. No alt allele here.
  • NoCall → genotype ./. — too little or too ambiguous evidence to make a confident genotype.
0/0 and ./. rows are therefore not variants carried by this individual — they're sites the caller looked at and ruled out or couldn't resolve. Tick “Hide 0/0 and ./.” above to drop them and see only genotypes with an alternate allele. (The Zygosity → “HET or HOM” option does the same thing.)
Column guide — what each column means
Gene
Gene symbol the variant overlaps (VEP). Blank for intergenic positions.
Location
Chromosome and position on GRCh38; the cell links out to the variant in gnomAD.
Protein
Protein-level change in HGVS notation (p.…), for coding variants.
Consequence
Predicted functional effect (VEP / Sequence Ontology term, e.g. missense_variant, stop_gained, intron_variant).
Impact
VEP severity class: HIGH (likely disruptive, e.g. loss-of-function), MODERATE (e.g. missense), LOW (e.g. synonymous), MODIFIER (non-coding / intronic).
Zygosity
This sample's genotype: HET = one alt copy (0/1), HOM = two alt copies (1/1); also the reference (0/0) and no-call (./.) states described above.
CADD
CADD Phred deleteriousness score. ~10 ≈ top 10% most deleterious genome-wide, ~20 ≈ top 1%, ≥30 ≈ top 0.1%.
REVEL
Ensemble missense-pathogenicity score, 0–1; higher = more likely pathogenic (≥0.5 is a common flag). Missense variants only.
SpliceAI
Maximum SpliceAI delta score, 0–1, for predicted splice disruption. ≥0.2 possible, ≥0.5 likely splice-altering.
SIFT
SIFT missense prediction: “deleterious” vs “tolerated” (lower score = more damaging).
PolyPhen
PolyPhen-2 missense prediction: “probably/possibly damaging” vs “benign” (higher score = more damaging).
gnomAD
Population allele frequency (gnomAD genomes). Blank ≈ not observed in gnomAD (novel / very rare).
ClinVar
ClinVar clinical-significance assertion (e.g. pathogenic, likely_pathogenic, benign, conflicting), where the position is in ClinVar.
dbSNP / IDs
Known variant identifiers (dbSNP rsIDs, linked to dbSNP). Blank = not previously catalogued.
Location (hg19)
Same locus lifted to GRCh37/hg19, for cross-referencing hg19-based resources (e.g. ALSoD); links to the UCSC browser (hg19). Shown only when the Parquet carries a pos_hg19 column — see the README.

Tandem Repeat Expansions — TRGT (6 loci)

Repeat-locus genotypes from PacBio TRGT (step 03b). Unlike SNVs and SVs, tandem repeat expansions (e.g. C9orf72, HTT, the SCA loci) cannot be reliably called by standard small-variant or SV callers; TRGT is the only reliable caller for these loci. Click a row to view the waterfall plot of individual reads at that locus.

Field guide — what each value on a locus card means
Locus
Tandem-repeat locus ID (TRID) from the TRGT repeat catalog.
Status badge
Threshold classification: pathogenic, intermediate / risk, normal, or “no threshold defined” (locus genotyped but no clinical cutoff is catalogued).
Location
Chromosome and position of the repeat on GRCh38.
Motif
Repeat unit sequence(s) genotyped at this locus (e.g. GGGGCC for C9orf72).
GT
Genotype — which catalogued alleles the two haplotypes correspond to.
Repeats
Motif copy count per allele (MC) — the actual repeat sizes; this is what thresholds are applied to.
Depth
Number of spanning reads supporting each allele (SD); more reads = more reliable.
Allele lengths (bp)
Total length of each repeat allele in base pairs (AL).
Allele purity
Fraction of each allele matching the canonical motif (AP), 0–1; lower = interrupted / impure repeat.
CpG methylation
Mean CpG methylation per allele (AM), when available — relevant for loci like C9orf72 and FMR1.
Confidence interval
Repeat-length confidence interval per allele (ALLR).

All Structural Variants — Full Genome (…)

Every PBSV structural-variant call (DEL, DUP, INV, INS, BND, CNV). Same in-browser DuckDB engine as the All-Variants tab. Use the type/size filters to narrow down — most SVs in a WGS are common, short, and non-coding. Note: gene/consequence columns are populated only when step 06 ran VEP on the SV VCF; un-annotated SVs still appear and are searchable by position, type, and size.

Column guide — what each column means
Location
Chromosome and span (start–end) of the structural variant on GRCh38.
Type
SV class: DEL (deletion), DUP (duplication), INV (inversion), INS (insertion), BND (breakend / translocation), CNV (copy-number).
Size (bp)
Length of the structural variant in base pairs.
Gene
Gene the SV overlaps (only populated when VEP annotated the SV VCF).
Consequence
Predicted functional effect of the SV (VEP term), where annotated.
Impact
VEP severity class: HIGH / MODERATE / LOW / MODIFIER, where annotated.
Zygosity
This sample's genotype: HET = one alt copy, HOM = two alt copies (0/0 = reference, ./. = no-call).
DP
Total read depth (read support) at the SV locus.
AD (ref,alt)
Allelic depth — reads supporting the reference allele vs the alternate (SV) allele.
Qual
PBSV variant quality score; higher = more confident.
Filter
Caller FILTER status; PASS = passed PBSV's internal filters.
Precise
Whether breakpoints are base-pair precise (✓) or approximate / imprecise (~).
ID
PBSV's internal structural-variant identifier.
